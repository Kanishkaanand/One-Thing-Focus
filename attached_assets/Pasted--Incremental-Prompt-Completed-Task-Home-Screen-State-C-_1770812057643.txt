## Animation Sequence on First View

When the user first sees this state after completing their task (right after the celebration animation from the completion flow), play this sequence:

1. **0ms:** Screen transitions to State C layout
2. **0-600ms:** Checkmark draws on the card (SVG stroke animation)
3. **300ms:** Card border fades to sage green
4. **600ms:** Soft radial pulse on the checkmark (one pulse, not repeating)
5. **900ms:** Completion message fades in above the card
6. **1200ms:** Reflection summary fades in below the card
7. **1500ms:** "See you tomorrow âœ¨" fades in at the bottom

Total sequence: ~1.5 seconds. Should feel unhurried and satisfying.

On subsequent app opens the same day, skip the animation â€” show everything already in place, no re-animation.

## Edge Case: Multiple Tasks (Level 2 or 3)

If the user is at Level 2 or 3 and has completed SOME but not all tasks:
- Do NOT show State C
- Stay in State B (active task view)
- Completed tasks show with a subtle checkmark and muted styling
- Remaining tasks stay active with "Mark Complete" button
- Only transition to State C when ALL tasks for the day are done
