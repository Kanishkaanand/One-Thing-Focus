## Design Direction

### Aesthetic: Warm Minimalism meets Calm Technology

This app should feel like a **deep breath** â€” not another noisy productivity tool. Think: the calmness of a Muji store meets the warmth of a handwritten note.

### Design Principles
- **Ultra-light UI.** Lots of whitespace. No clutter. Every screen should feel like it has room to breathe.
- **Warm, not cold.** Avoid sterile grays and blues. Use warm neutrals (soft cream, warm stone, muted terracotta) with one accent color (a soft golden amber or warm coral).
- **Gentle motion.** Subtle fade-ins, soft spring animations on task cards, smooth transitions between screens. Nothing aggressive or bouncy â€” think slow inhale/exhale rhythm.
- **Typography-led.** Use a distinctive serif or soft rounded font for headings (like "Fraunces", "Lora", or "Nunito") paired with a clean sans-serif body font (like "DM Sans" or "Plus Jakarta Sans"). The heading font should feel friendly and human, not corporate.
- **No harsh reds or failure states.** Even "missed" days should feel gentle â€” use muted tones, never alarming colors.
- **Micro-celebrations.** When a task is completed, show a delightful but subtle animation â€” a gentle confetti of dots, a soft glow, or a satisfying checkmark animation. Make completion feel rewarding without being childish.

## Core Logic & Data Model

### Level System

```
Level 1: 1 task per day (starting level)
Level 2: 2 tasks per day (unlocked after 7 consecutive completed days at Level 1)
Level 3: 3 tasks per day (unlocked after 7 consecutive completed days at Level 2)
```

**Upgrade Rule:** Complete ALL tasks for 7 consecutive days at current level â†’ level up.

**Downgrade Rule:** If at Level 2 or 3 and user misses completing all tasks for a day â†’ downgrade to previous level. Streak for current level resets to 0.

**Important:** Level 1 users do NOT get downgraded â€” they just lose their streak. We never punish beginners.

### Data Model (AsyncStorage)

```typescript
// User Profile
interface UserProfile {
  name: string;
  createdAt: string; // ISO date
  currentLevel: 1 | 2 | 3;
  currentLevelStreak: number; // consecutive days completed at current level
  longestStreak: number;
  totalTasksCompleted: number;
  reminderEnabled: boolean;
  reminderTime: string; // "09:00"
}

// Daily Entry
interface DailyEntry {
  date: string; // "2026-02-11"
  tasks: TaskItem[];
  reflection?: {
    mood: 'energized' | 'calm' | 'neutral' | 'tough';
    note?: string;
  };
  completed: boolean; // all tasks done
  levelAtTime: 1 | 2 | 3;
}

// Individual Task
interface TaskItem {
  id: string;
  text: string;
  createdAt: string;
  completedAt?: string;
  proof?: {
    type: 'photo' | 'screenshot' | 'document';
    uri: string;
  };
  isCompleted: boolean;
}
```

### End of Day Logic

If the day ends and tasks are not completed:
- Do NOT show any negative messaging
- Next morning, greet the user warmly and show:
  > "Yesterday didn't go as planned â€” that's completely okay. Every day is a fresh start. What's your one thing today?"
- If the user was at Level 2 or 3, show the downgrade gently:
  > "You're back to [X] task(s) per day. No pressure â€” just focus on showing up."
- The missed day appears in calendar with a neutral/soft indicator, NOT a failure mark

## Copy & Tone of Voice

The app's written voice should be: **warm, encouraging, and human.** Like a supportive friend, not a drill sergeant.

- Never say: "You failed", "Missed", "Incomplete", "0 tasks done"
- Instead say: "Rest day", "Tomorrow's a fresh start", "Take it easy"
- Streak messages should build excitement without pressure:
  - Day 1: "And so it begins."
  - Day 3: "Three days strong. You're finding your rhythm."
  - Day 7: "A whole week! You've earned this."
  - Day 14: "Two weeks of showing up. That's not luck â€” that's you."
  - Day 30: "30 days. You've built something real."
